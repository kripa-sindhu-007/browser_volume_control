# Security Policy

## Reporting A Vulnerability

Please do not open a public GitHub issue for security vulnerabilities.

Instead, report the issue privately using GitHub private vulnerability reporting if it is enabled for the repository. If private reporting is not available, contact the maintainer directly through their GitHub profile.

## What To Include

- A clear description of the vulnerability.
- Steps to reproduce.
- Affected browser version.
- Affected extension version or commit.
- Any proof-of-concept details needed to validate the issue.

## Project Security Goals

- Avoid unnecessary permissions.
- Do not execute remote JavaScript.
- Do not collect analytics or telemetry.
- Keep user preferences local to the browser.
- Treat all web page content as untrusted.

## Supported Versions

This project is in early development. Security fixes target the latest code on the default branch.
